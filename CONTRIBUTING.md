# Contributing to ComputerPets

Thank you for your interest in contributing to **ComputerPets** (EnterprisePet Backend)!

We welcome contributions of all kinds — code, documentation, bug reports, feature ideas, and design discussions.

## Quick Links

- **[Full Contribution Guidelines](docs/CONTRIBUTING.md)** — Detailed process, coding standards, and pull request checklist.
- **[Documentation Index](docs/README.md)** — Architecture, setup, and API docs.
- **[Architecture Overview](docs/ARCHITECTURE.md)** — Deep dive into the system design.

## How to Contribute

1. **Fork** the repository and create a branch from `main`.
2. **Make your changes** following the guidelines in `docs/CONTRIBUTING.md`.
3. **Update documentation** — especially `docs/ARCHITECTURE.md` for any architectural changes.
4. **Open a Pull Request** using the PR template.
5. **Be responsive** to review feedback.

## Development Setup

See the [Setup Guide](docs/SETUP.md) for instructions on building and running the project locally (includes Windows/PowerShell instructions).

## Code of Conduct

Please be respectful and constructive. We aim to maintain a welcoming environment for all contributors.

## Questions?

Open a [GitHub Discussion](https://github.com/your-org/ComputerPets/discussions) or an issue with the `question` label.
